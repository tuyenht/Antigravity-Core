## Reporting a Vulnerability

If you discover a security vulnerability, **please do NOT open a public issue.**

### How to Report

1. **Email**: Open a **private** security advisory on [GitHub](https://github.com/tuyenht/Antigravity-Core/security/advisories/new)
2. **Include**: Description, steps to reproduce, affected files/components, potential impact

### Response Timeline

| Stage | Timeline |
|-------|----------|
| Acknowledgment | Within 48 hours |
| Assessment | Within 5 business days |
| Fix (if confirmed) | Within 14 business days |
| Public disclosure | After fix is released |

## Scope

This policy covers the `.agent/` system including:
- Agent definitions, skills, and workflows
- Automation scripts (`scripts/`)
- Configuration files (`GEMINI.md`, `project.json`)
- Installation scripts (`install-global.ps1`)

## Recognition

Security researchers who responsibly disclose vulnerabilities will be credited in the CHANGELOG and release notes.
