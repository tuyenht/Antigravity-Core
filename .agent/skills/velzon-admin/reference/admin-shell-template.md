## Header Spec

### Component Order (left → right)

```
┌─────────────────────────────────────────────────────────────────────────┐
│ [☰ Hamburger]  [🔍 Search...]              [🇺🇸] [☾] [⛶] [🔔3] [👤▾] │
└─────────────────────────────────────────────────────────────────────────┘
  ↑               ↑                            ↑   ↑   ↑    ↑     ↑
  Toggle         SearchOption            Lang Dark Full Notif Profile
  Sidebar                                Drop  Mode screen Drop  Drop
```

### 1. Sidebar Toggle (Hamburger)

```tsx
<button
    onClick={toggleMenuBtn}
    type="button"
    className="btn btn-sm px-3 fs-16 header-item vertical-menu-btn topnav-hamburger"
    id="topnav-hamburger-icon">
    <span className="hamburger-icon">
        <span></span>
        <span></span>
        <span></span>
    </span>
</button>
```

**Behavior:**
- Desktop (>1025px): Toggles `data-sidebar-size` between `'lg'` and `'sm'` (icon-only)
- Tablet (768–1025px): Toggles `data-sidebar-size` between `'sm'` and `''`
- Mobile (<767px): Adds/removes `vertical-sidebar-enable` class on `body`

### 2. SearchOption

**Source:** `source/react-ts/header-components/SearchOption.tsx` (205 lines)

- Desktop: Input field with `Search...` placeholder, always visible
- Mobile: Icon button that opens dropdown with search input
- Searches through menu items from `LayoutMenuData`
- Shows recent searches and matching results

### 3. LanguageDropdown

**Source:** `source/react-ts/header-components/LanguageDropdown.tsx` (67 lines)

- Shows current language flag icon (🇺🇸 for EN)
- Dropdown with flag + language name for each supported locale
- Stores selection in `localStorage('I18N_LANGUAGE')`
- **Admin shell uses i18next** (different from auth pages which use cookie-based locale)

### 4. LightDark Toggle

**Source:** `source/react-ts/header-components/LightDark.tsx` (25 lines)

```tsx
<button
    onClick={() => onChangeLayoutMode(mode)}
    type="button"
    className="btn btn-icon btn-topbar btn-ghost-secondary rounded-circle light-dark-mode">
    <i className='bx bx-moon fs-22'></i>
</button>
```

- Toggles between `LAYOUT_MODE_TYPES.LIGHTMODE` and `DARKMODE`
- Updates `data-layout-mode` attribute on `<html>`

### 5. FullScreenDropdown

- Toggle fullscreen via `document.documentElement.requestFullscreen()` / `document.exitFullscreen()`
- Icon changes: `ri-fullscreen-line` → `ri-fullscreen-exit-line`

### 6. NotificationDropdown

**Source:** `source/react-ts/header-components/NotificationDropdown.tsx`

- Bell icon with badge count (e.g., `3`)
- Dropdown with tabs: All / Messages / Alerts
- Each notification: avatar + title + description + timestamp
- "View All Notifications" link at bottom

### 7. ProfileDropdown

**Source:** `source/react-ts/header-components/ProfileDropdown.tsx` (117 lines)

- Avatar (32×32 rounded) + user name + role subtitle (e.g., "Founder")
- Dropdown menu items: Profile, Messages, Taskboard, Help, Settings, Lock screen, Logout
- Name from session, role from user permissions

## Sidebar Spec

### Structure

```
┌─────────────────────────────┐
│  [Logo]            [● hover]│ ← Logo + sm-hover toggle
│─────────────────────────────│
│  [Avatar] Name      Online  │ ← Sidebar user profile
│─────────────────────────────│
│  MENU                       │ ← Category label
│  ⊙ Dashboard          ▾    │ ← Active item with chevron
│    └─ Analytics             │
│    └─ CRM                   │
│    └─ Ecommerce             │
│  ⊙ User Management    ▾    │
│    └─ All Users             │
│    └─ Add User              │
│  ⊙ Roles & Permissions     │
│  ⊙ Settings                │
│─────────────────────────────│
│  PAGES                      │ ← Second category
│  ⊙ Authentication     ▾    │
│  ⊙ Pages              ▾    │
└─────────────────────────────┘
```

### Key Features

1. **Logo area**: Shows `logo-light.png` by default, `logo-sm.png` when sidebar collapsed
2. **Sidebar user**: Avatar + name + online status (optional, controlled by layout settings)
3. **Multi-level menu**: Supports 3+ levels with smooth accordion animations
4. **Active state**: Current page highlighted with active class + left border glow
5. **Badges**: Support for `<span className="badge">New</span>` on menu items
6. **Permission filtering**: Menu items filtered by user permissions via `filterMenu()`
7. **Collapse behavior**: `data-sidebar-size="sm"` shows only icons with hover tooltip
8. **SimpleBar scrolling**: Sidebar uses SimpleBar for custom scrollbar styling

### Menu Data Format

```tsx
// LayoutMenuData.tsx
export interface MenuItem {
    id: string;
    label: string;
    icon: string;                    // RemixIcon class
    link: string;
    permission?: string;             // Required permission
    badge?: { text: string; class: string }; // Optional badge
    subItems?: MenuItem[];           // Nested items (unlimited depth)
    isHeader?: boolean;              // Category separator (e.g., "MENU", "PAGES")
}
```

## Theme Customizer (RightSidebar) Spec

### Trigger

Floating gear icon button, fixed position on right edge of screen:

```css
/* Floating settings button */
.btn-setting {
    position: fixed;
    right: 0;
    top: 50%;
    transform: translateY(-50%);
    z-index: 1050;
    /* Animated rotation on hover */
}
```

### Customizer Drawer Options

| Section | Options | Data Attribute |
|---------|---------|---------------|
| **LAYOUT** | Vertical, Horizontal, Two Column | `data-layout` |
| **COLOR SCHEME** | Light, Dark | `data-layout-mode` |
| **LAYOUT POSITION** | Fixed, Scrollable | `data-layout-position` |
| **TOPBAR COLOR** | Light, Dark | `data-topbar` |
| **SIDEBAR SIZE** | Default, Compact, Small (Icon), Small Hover | `data-sidebar-size` |
| **SIDEBAR VIEW** | Default, Detached | `data-layout-style` |
| **SIDEBAR COLOR** | Light, Dark, Gradient (4 options) | `data-sidebar` |
| **SIDEBAR IMAGE** | None, Image 1-4 | `data-sidebar-image` |

### Implementation Pattern

## Layout Orchestrator (index.tsx) Spec

The root `Layout` component wires everything together:

```tsx
<div id="layout-wrapper">
    <Header
        headerClass={headerClass}
        layoutModeType={layoutModeType}
        onChangeLayoutMode={onChangeLayoutMode}
    />
    <Sidebar layoutType={layoutType} />
    <div className="main-content">
        {children}
        <Footer />
    </div>
</div>
<RightSidebar />  {/* Offcanvas, outside layout-wrapper */}
```

### Layout State Management

For **Next.js** (no Redux), use React Context + localStorage:

```tsx
interface LayoutState {
    layoutType: 'vertical' | 'horizontal' | 'twocolumn';
    layoutModeType: 'light' | 'dark';
    layoutPositionType: 'fixed' | 'scrollable';
    topbarTheme: 'light' | 'dark';
    sidebarSize: 'lg' | 'sm' | 'sm-hover' | 'md';
    sidebarColor: 'light' | 'dark' | 'gradient-1' | 'gradient-2' | 'gradient-3' | 'gradient-4';
    sidebarVisibility: 'show' | 'hidden';
}
```

- **Persist to localStorage** so user preferences survive page refresh
- **Apply via `data-*` attributes** on `<html>` element (Velzon CSS reads these)

## Adaptation Notes (Per Framework)

### Next.js (App Router)

1. **No Redux** — Use React Context (`LayoutContext`) + `useReducer` instead of Redux slices
2. **No react-router-dom** — Use `next/link` for `<Link>` and `next/navigation` for `usePathname()`
3. **No reactstrap** — Use headless UI or custom dropdown components (or keep reactstrap as dependency)
4. **Images** — Use `next/image` or static imports instead of require paths
5. **Layout** — Component lives in `app/admin/(dashboard)/layout.tsx`, wrapping all dashboard pages
6. **SimpleBar** — `simplebar-react` works in Next.js (client component)

### Key Principle

> The AI MUST copy the actual source files from `source/react-ts/` and adapt import paths/routing.
> It MUST NOT generate a simplified monolith that omits features.
> Each component file MUST be preserved as a separate file for maintainability.
