# Environment Configuration Template

> **Applies to:** ALL projects generated by `/create-admin`
> **Source:** Best practices from `baoson-platform-core` + OWASP standards

---

## Laravel `.env.example`

> [!IMPORTANT]
> `/create-admin` MUST generate `.env.example` with these defaults.
> `php artisan key:generate` MUST be run during scaffold to populate `APP_KEY`.

```env
# ============================================================================
# APPLICATION
# ============================================================================
APP_NAME="Admin Panel"
APP_ENV=local
APP_KEY=
APP_DEBUG=true
APP_URL=http://localhost
ADMIN_PREFIX=admin

# ============================================================================
# LOCALIZATION (i18n-ready from start)
# ============================================================================
APP_LOCALE=en
APP_FALLBACK_LOCALE=en
APP_FAKER_LOCALE=en_US

# ============================================================================
# MAINTENANCE
# ============================================================================
APP_MAINTENANCE_DRIVER=file

# ============================================================================
# SERVER
# ============================================================================
PHP_CLI_SERVER_WORKERS=4

# ============================================================================
# SECURITY (OWASP Compliant)
# ============================================================================
BCRYPT_ROUNDS=12

# ============================================================================
# LOGGING
# ============================================================================
LOG_CHANNEL=stack
LOG_STACK=single
LOG_DEPRECATIONS_CHANNEL=null
LOG_LEVEL=debug

# ============================================================================
# DATABASE (SQLite for local dev, PostgreSQL for production)
# ============================================================================
DB_CONNECTION=sqlite
# DB_HOST=127.0.0.1
# DB_PORT=5432
# DB_DATABASE=admin_panel
# DB_USERNAME=root
# DB_PASSWORD=

# ============================================================================
# SESSION & CACHE
# ============================================================================
SESSION_DRIVER=database
SESSION_LIFETIME=120
SESSION_ENCRYPT=false
SESSION_PATH=/
SESSION_DOMAIN=null

CACHE_STORE=database

# ============================================================================
# QUEUE & BROADCAST
# ============================================================================
QUEUE_CONNECTION=database
BROADCAST_CONNECTION=log

# ============================================================================
# FILESYSTEM
# ============================================================================
FILESYSTEM_DISK=local

# ============================================================================
# MAIL (log driver = safe for dev, no accidental emails)
# ============================================================================
MAIL_MAILER=log
MAIL_SCHEME=null
MAIL_HOST=127.0.0.1
MAIL_PORT=2525
MAIL_USERNAME=null
MAIL_PASSWORD=null
MAIL_FROM_ADDRESS="admin@example.com"
MAIL_FROM_NAME="${APP_NAME}"

# ============================================================================
# REDIS (uncomment when needed)
# ============================================================================
# REDIS_CLIENT=phpredis
# REDIS_HOST=127.0.0.1
# REDIS_PASSWORD=null
# REDIS_PORT=6379

# ============================================================================
# VITE (bridge to frontend)
# ============================================================================
VITE_APP_NAME="${APP_NAME}"
```

## Next.js `.env.local`

> [!IMPORTANT]
> `/create-admin` MUST generate `.env.local` (gitignored) + `.env.example` (committed).
> NextAuth `SECRET` MUST be generated via `npx auth secret` or `openssl rand -base64 32`.

```env
# ============================================================================
# APPLICATION
# ============================================================================
NEXT_PUBLIC_APP_NAME="Admin Panel"
NEXT_PUBLIC_APP_URL=http://localhost:3000
ADMIN_PREFIX=admin

# ============================================================================
# AUTHENTICATION (NextAuth.js / Auth.js)
# ============================================================================
AUTH_SECRET=                          # REQUIRED: npx auth secret
AUTH_URL=http://localhost:3000

# OAuth Providers (uncomment as needed)
# GOOGLE_CLIENT_ID=
# GOOGLE_CLIENT_SECRET=
# GITHUB_CLIENT_ID=
# GITHUB_CLIENT_SECRET=

# ============================================================================
# DATABASE
# ============================================================================
# SQLite (local dev — default)
DATABASE_URL="file:./dev.db"

# PostgreSQL (production — uncomment)
# DATABASE_URL="postgresql://user:password@localhost:5432/admin_panel?schema=public"

# ============================================================================
# MAIL (API-based for Next.js)
# ============================================================================
# RESEND_API_KEY=
# MAIL_FROM="Admin Panel <admin@example.com>"

# ============================================================================
# SECURITY
# ============================================================================
BCRYPT_ROUNDS=12
```

## Key Design Decisions

| Decision | Rationale |
|----------|-----------|
| `BCRYPT_ROUNDS=12` | OWASP minimum 10, 12 is sweet spot (perf vs security) |
| `SESSION_DRIVER=database` | Survives server restarts, works in multi-server |
| `CACHE_STORE=database` | No Redis dependency for starter projects |
| `MAIL_MAILER=log` | Prevents accidental emails during dev |
| `PHP_CLI_SERVER_WORKERS=4` | Multi-request handling for `php artisan serve` |
| `QUEUE_CONNECTION=database` | Enables async jobs without Redis |
| `DB_CONNECTION=sqlite` | Zero-config, single file, perfect for dev/demo |
| `ADMIN_PREFIX=admin` | Configurable admin route prefix |
| `APP_LOCALE=en` | i18n-ready from day one |
