# Quarterly Maintenance Tasks

**Purpose:** Major system updates and strategic planning  
**Schedule:** Every 3 months (Jan, Apr, Jul, Oct)  
**Time Commitment:** 4-6 hours

### Framework Version Bumps

- [ ] **Laravel Major Version**
  - Check for Laravel 13, 14, etc.
  - Review migration guides
  - Test breaking changes
  - Update all conventions
  - Update examples and patterns

- [ ] **React Major Version**
  - Check for React 20, 21, etc.
  - Review new features
  - Update compiler docs
  - Refresh component patterns

- [ ] **Next.js Major Version**
  - Check for Next.js 17, 18, etc.
  - Review new App Router features

  - Check for TypeScript 6, 7, etc.
  - Review type system changes
  - Update tsconfig recommendations

- [ ] **Mobile Frameworks**
  - React Native major updates
  - Flutter major updates
  - Platform-specific changes

### Tasks:
- [ ] Identify all major version changes (last 3 months)
- [ ] Prioritize by impact and adoption
- [ ] Create upgrade guides
- [ ] Update all affected conventions
- [ ] Test examples and code snippets
- [ ] Update tech radar adoption dates

### Output:
- [ ] Major version upgrade log

## 2. Skill Ecosystem Review üéØ

### Complete Skill Audit

- [ ] **Usage Analysis (last 3 months)**
  - Skills used frequently (keep & enhance)
  - Skills used occasionally (evaluate)
  - Skills never used (retire or improve)

- [ ] **Quality Assessment**
  - Review all skill SKILL.md files
  - Check for outdated information
  - Verify examples still work
  - Update dependencies

- [ ] **Gap Analysis**
  - New technologies emerged?
  - Missing skill areas identified?
  - User requests not covered?

- [ ] **Skill Creation**
  - Create 3-5 new high-impact skills
  - Based on usage patterns
  - Based on strategic direction

- [ ] **Skill Retirement**
  - Archive deprecated skills
  - Remove unmaintained skills
  - Consolidate duplicate skills

### Output:
- [ ] Skill ecosystem report
- [ ] 3-5 new skills created
- [ ] Deprecated skills archived
- [ ] Updated skills-gap-analysis.md

## 3. Agent Restructuring ü§ñ

### Comprehensive Agent Review

- [ ] **Activation Patterns**
  - Which agents activated most? (keep)
  - Which never activated? (remove or merge)
  - Missing agent roles?

- [ ] **Role Clarity**
  - Review all agent descriptions
  - Ensure no overlap
  - Clear responsibility boundaries

- [ ] **Knowledge Base Updates**
  - Add successful patterns to agents
  - Update skill lists
  - Refresh examples

- [ ] **New Agent Creation**
  - Identify missing specialist roles
  - Create 1-2 new agents if needed
  - Based on project types

- [ ] **Agent Consolidation**
  - Merge similar agents
  - Remove unused agents
  - Keep count manageable (10-15)

### Output:
- [ ] Agent activation report
- [ ] Updated agent definitions
- [ ] New agents created (if needed)
- [ ] Deprecated agents archived
